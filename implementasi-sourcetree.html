Implementasi : Sourcetree

Management Git.

Download dan pasang SourceTree kamu.

Sudahkah kamu menggunakan SourceTree hari ini? haha. Jika kamu belum mengenal SourceTree, aplikasi ini adalah Git dan Mercurial Client yang dibagikan secara gratis hanya untuk Windows dan Mac, jika kamu pengguna GNU/Linux jangan bersedih hati, kamu bisa menggunakan GitEye sebagai Git Clientnya. Download SourceTree kamu di https://www.sourcetreeapp.com/ dan install sesuai instruksi dari SourceTree tersebut.

Setting SourceTree

Jika kamu sudah menginstall SourceTree, sebaiknya kamu mengatur SSH Key yang sudah kamu punya pada sebelumnya. Kamu bisa mengatur SSH Client Configuration dengan file id_rsa.pub yang kamu miliki.

Clone Repo dari GitLab

Saatnya clone repo kamu yang sudah disetup sebelumnya dengan menggunakan SourceTree, pada materi sebelumnya terdapat url SSH pada halaman dashboard project, dengan menggunakan url SSH kita mengontrol repo yang ada di GitLab. Source Path / URL kamu isi dengan url SSH dari halaman project sebelumnya, dan Destination Path kamu isi dengan path folder dimana kamu akan menyimpan repo tersebut. Dan pada bagian Bookmark, form ini akan terisi otomatis, kamu bisa mengganti Name sesuai dengan yang kamu inginkan.

Tampilan Repo yang sudah diclone

Jika kamu melakukan clone terhadap project yang baru memang tidak akan menampilkan apa-apa karena belum ada aktivitas dari project tersebut. Tapi nantinya jika kamu melakukan clone terhadap project yang sudah berjalan lama, akan menampilkan timeline commit terdahulu, dan akan menampilkan branch-branch.

Buat file readme

Sebagai awal dari project biasanya kita membuat file readme, ritual awal ini bisa memberikan informasi kepada development lain untuk mengikuti instruksi didalam readme tersebut, tapi disini kita membuat readme sebagai contoh bagaimana proses remote repo berlangsung.

First Commit

Setelah kamu membuat file readme sebelumnya, kita kembali ke aplikasi Git Client SourceTree, SourceTree akan mendeteksi file mana saja yang dirubah oleh user pada bagian Working Copy, lalu kita akan melakukan staging file mana saja yang akan dipush, dalam hal ini kita akan melakukan staging terhadap file Readme.md. Setelah melakukan staging, kamu bisa menambahkan commit yang inputannya ada dipaling bawah, commit ini berguna untuk memberikan komentar terhadap file yang kita rubah.

Network Repo

Setelah melakukan commit, network akan menampilkan hasil commit kita yang masih ada di lokal.

Push

Setelah semua pekerjaan kamu dirasa sudah cukup dan orang lain harus melihat hasil pekerjaan kamu, kamu harus melakukan PUSH. Jadi ibarat kamu ngumpulin tugas, kalau udah beres pasti kamu harus mengirimkannya kan? nah fasilitas PUSH ini adalah bagian proses mengumpulkan tugas tersebut. Jadi nantinya pekerjaan kamu akan tersimpan di Git dan teman kamu dapat melihat dan menggabungkannya.

Push Success

Ingat, Push harus sampai berhasil ya kaya gambar yang di atas.

Edit di Gitlab

Kamu bisa langsung edit file nya di gitlab lhooo, biasa nya kalo lagi Urgent saat kamu lagi ga buka laptop kamu cuma pegang smartphone kesayangan kamu. Kamu bisa edit Codingan kamu melalui Gitlab.

Edit readme di Gitlab

Pull Repo

Pada pekerjaan team work kita harus bisa menginformasikan langkah-langkah penting untuk anggota team, terutama jika pada saat itu ada anggota baru di team kamu. Anggota team kamu hanya cukup membaca Readme lalu dia bisa bergabung dalam pekerjaan di team kamu. Nah itu lah pentingnya Readme.

Jika ada perubahan pada branch, kamu sudah seharusnya menggunakan fitur Pull pada aplikasi SourceTree, dengan menggunakan Pull kamu akan mengupdate kodingan di branch Master dengan kodingan terakhir kali di push ke branch tersebut.

Sukses pull

Jika kamu tidak memilii konflik source dengan hasil pull, maka akan menampilkan status sukses setelah melakukan Pull.

Network Repo

Setelah berhasil melakukan pull, maka network pada aplikasi SourceTree akan menampilkan commit terakhir yang di push pada repo server.
