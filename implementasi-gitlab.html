Implementasi : Gitlab

GitLab alternatif repository kamu.

Sudah menjadi hal umum jika menaruh codingan kamu di github, gitlib, bitbucket dan server-server lainnya. GitLab bisa jadi alternatif kamu untuk menaruh kodingan kamu selain tempat-tempat yang disebutkan sebelumnya. Kelebihan GitLab daripada server lainnya adalah kamu bisa buat sebanyak mungkin private project di online server dan kali ini kita akan menggunakan GitLab sebagai server repository-nya, biar kamu lebih mengenal GitLab lebih jauh.

Daftar GitLab, sekarang!

Daftar

Gampang banget buat join di gitlab, bisa pake akun google, twitter, github, bitbucket kamu. Tapi jika kamu belum punya akun diatas kamu masih tetep bisa daftar dengan mengisikan form pendaftaran yang letaknya dibawah halaman.

Ubah profile

Setelah melalui tahap pendaftaran super gampang diatas, kamu harus tetap mengisikan email kamu pada halaman profile, untuk membantu urusan privilege, semua aktivitas yang berkaitan dengan project akan dikirimkan ke e-mail kamu, agar project kamu aman jika terjadi sesuatu yang tidak diinginkan.

Setup Password

Jika kamu mendaftar dengan akun terkait (seperti google, twitter, github, dan bitbucket), kamu harus melakukan setup password untuk akun kamu di GitLab, karena jika akun kamu belum melakukan setup password kamu tidak bisa melakukan pull/push project karena untuk melakukan perintah tersebut kamu membutuhkan password sebagai privilege kamu sebagai owner project.

Download GIT

Jika kamu menggunakan Windows, kamu membutuhkan Git sebagai base mesin pengatur repository yang kamu miliki, dan Git Bash yang dimiliki oleh Git, layaknya shell-command di linux, kamu bisa menggunakan perintah-perintah sederhana yang di linux jika menggunakan Git Bash ini dan kita membutuhkan Git Bash untuk melakukan Generate SSH Keys.

Generate SSH Keys

Kita membutuhkan SSH Keys sebagai kode pengenal antara komputer kita dan GitLab, sebelum melakukan generate SSH key sebaiknya kamu mengecek file id_rsa.pub di C:/Users/(username)/.ssh/ di Windows dan ~/.ssh/ di Linux, jika kamu sudah memiliki file tersebut kamu dapat melewati langkah generate SSH key.

Untuk melakukan generate SSH keys kamu harus mengikuti perintah dibawah ini.

ssh-keygen -t rsa -C "emailkamu@example.com"

Setelah melakukan generate SSH keys, kamu dapat mengcopy isi dari file id_rsa.pub. Kamu bisa menggunakan command yang biasa digunakan tergantung jenis sistem operasi yang digunakan.

Windows :

clip < C:/Users/(username)/.ssh/id_rsa.pub

Mac :

pbcopy < ~/.ssh/id_rsa.pub

GNU/Linux :

xclip -sel clip < ~/.ssh/id_rsa.pub

Halaman SSH Keys

Pada halaman SSH Keys, user dapat menambahkan hasil Generate SSH Keys dari komputer dan melihat seluruh list SSH Keys yang telah didaftarkan ke GitLab. Untuk menambahkan SSH Keys user dapat menekan tombol ADD SSH KEY.

Menambahkan SSH Key

Pada gambar diatas akan menampilkan form Key dan Title, anda dapat menambahkan hasil generate SSH key pada form Key dan Title untuk memberikan keterangan misalkan "Laptop Dell".

Dashboard Project

Setelah kamu melalui semua step diatas, akhirnya kamu sampai dititik untuk menambahkan project. Kamu bisa menambahkan project untuk perorangan, membuat group, maupun mencari project orang lain yang bersifat public. Disini kita akan mencoba untuk membuat group, yang isinya bisa diisi banyak project dan orang-orang yang dapat kita undang.

Activity merupakan halaman untuk menampilkan aktivitas dari project yang sedang dikerjakan. Bagaikan timeline jejaring sosial kamu di Facebook/Twitter, Activity ini merupakan "jejaring sosial" kamu dalam hal project hehe.

Group merupakan halaman yang menampilkan kumpulan group yang diikuti/dibuat oleh kamu.

Issues jika kamu berkolaborasi, kamu membutuhkan media untuk tanya jawab / memberikan perintah / memberikan penjelasan spesifikasi dari proyek yang dibangun maupun memaparkan bug yang ditemukan dalam proyek, Issues merupakan media yang dapat digunakan bagaikan forum-forum.

Snippets bagian ini bermanfaat untuk kamu yang ingin berkolaborasi bersama teman-teman hanya beberapa baris kode, sehingga dapat dengan mudah membacanya, biasanya snippet ini digunakan untuk berbagi kode SQL yang harus diujicoba bersama-sama, sehingga jika sudah cocok baru dimasukan ke dalam kode project.

Membuat Group

Pada gambar diatas, kamu harus mengisi url yang akan digunakan oleh group yang kamu buat, dan sedikit memberikan penjelasan pada bagan Detail agar kamu dan teman-teman kamu ingat untuk apa group ini dibuat hehe.

Dashboard Group

Hampir mirip dengan dashboard perorangan, dashboard group memiliki menu yang fungsinya sama seperti pada bagian dashboard perorangan. Bedanya ada menu Member dimana kamu bisa menambahkan anggota group yang dapat mengakses halaman group tersebut. Dan dengan menekan tombol NEW PROJECT kamu akan dipindahkan ke halaman membuat project.

Membuat Project Baru

Akhirnya kamu membuat project pertamamu di GitLab! Hal-hal yang penting kamu isikan adalah alamat project yang ingin kamu buat, Namespace adalah tempat kamu ingin untuk membuat project tersebut, bisa jadi group ataupun perorangan. Repository project GitLab dapat di-Import dari server repository lama kamu biasa gunakan. Description memudahkan kamu menjelaskan secara singkat tentang project yang dibuat, dan yang terakhir Visibility Level dimana kamu dapat merubah settingan project kamu agar dapat dilihat orang lain atau hanya teman-teman kamu saja maupun kamu seorang yang dapat mengakses project tersebut.

Dashboard Project

Tadaaaa! Repo pertama kamu di GitLab telah selesai dibuat, mudah bukan? Dashboard pada project memiliki kemiripan dengan dashboard lainnya, bedanya pada bagian tengah dashboard terdapat bagian url ssh dan https, jika kita menggunakan link https sebagai control repo kita maka akan selalu ditanya username dan password saat melakukan pull / push dan huja menggunakan link ssh kita hanya menggunakan SSH Key sebagai kode pengenal untuk mengontrol repo kita.

Files memberikan tampilan struktur dan list file yang ada di repo.

Commits tampilan semua hasil commits yang sudah dipush ke repo.

Network menampilkan jalur hasil commit yang sudah dipush, jika kamu bekerja dengan team dan membutuhkan pengetahuan siapa saja yang mengganti file yang kamu cari, dengan Network aku bisa membaca alur tersebut.

Graphs menampilkan banyaknya hasil commit yang di push ke repo ditampilkan dalam bentuk grafik, kita dapat melihat adanya peningkatan atau penuruan dari kinerja koding dengan hanya melihat halaman ini.

Wiki menampilkan wiki dokumentasi dari project, jika kamu baru bergabung kedalam sebuah project, dan kamu belum mengetahui apa-apa tentang project tersebut, dan readme hanya penjelasan singkat yang diberikan oleh developer sebelumnya, dengan Wiki kamu bisa mengetahui secara lengkap dokumentasi project tersebut.
