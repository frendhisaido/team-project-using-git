Implementasi : Gitlab

GitLab alternatif repository kamu.

Sudah menjadi hal umum jika menaruh codingan kamu di github, gitlib, bitbucket dan server-server lainnya. GitLab bisa jadi alternatif kamu untuk menaruh kodingan kamu selain tempat-tempat yang disebutkan sebelumnya. Kelebihan GitLab daripada server lainnya adalah kamu bisa buat sebanyak mungkin private project di online server dan kali ini kita akan menggunakan GitLab sebagai server repository-nya, biar kamu lebih mengenal GitLab lebih jauh.

Daftar GitLab, sekarang!

Daftar

Gampang banget buat join di gitlab, bisa pake akun google, twitter, github, bitbucket kamu. Tapi jika kamu belum punya akun diatas kamu masih tetep bisa daftar dengan mengisikan form pendaftaran yang letaknya dibawah halaman.

Ubah profile

Setelah melalui tahap pendaftaran super gampang diatas, kamu harus tetap mengisikan email kamu pada halaman profile, untuk membantu urusan privilege, semua aktivitas yang berkaitan dengan project akan dikirimkan ke e-mail kamu, agar project kamu aman jika terjadi sesuatu yang tidak diinginkan.

Setup Password

Jika kamu mendaftar dengan akun terkait (seperti google, twitter, github, dan bitbucket), kamu harus melakukan setup password untuk akun kamu di GitLab, karena jika akun kamu belum melakukan setup password kamu tidak bisa melakukan pull/push project karena untuk melakukan perintah tersebut kamu membutuhkan password sebagai privilege kamu sebagai owner project.

Download GIT

Jika kamu menggunakan Windows, kamu membutuhkan Git sebagai base mesin pengatur repository yang kamu miliki, dan Git Bash yang dimiliki oleh Git, layaknya shell-command di linux, kamu bisa menggunakan perintah-perintah sederhana yang di linux jika menggunakan Git Bash ini dan kita membutuhkan Git Bash untuk melakukan Generate SSH Keys.

Generate SSH Keys

Kita membutuhkan SSH Keys sebagai kode pengenal antara komputer kita dan GitLab, sebelum melakukan generate SSH key sebaiknya kamu mengecek file id_rsa.pub di C:/Users/(username)/.ssh/ di Windows dan ~/.ssh/ di Linux, jika kamu sudah memiliki file tersebut kamu dapat melewati langkah generate SSH key.

Untuk melakukan generate SSH keys kamu harus mengikuti perintah dibawah ini.

ssh-keygen -t rsa -C "emailkamu@example.com"

Setelah melakukan generate SSH keys, kamu dapat mengcopy isi dari file id_rsa.pub. Kamu bisa menggunakan command yang biasa digunakan tergantung jenis sistem operasi yang digunakan.

Windows :

clip < C:/Users/(username)/.ssh/id_rsa.pub

Mac :

pbcopy < ~/.ssh/id_rsa.pub

GNU/Linux :

xclip -sel clip < ~/.ssh/id_rsa.pub

Halaman SSH Keys

Pada halaman SSH Keys, user dapat menambahkan hasil Generate SSH Keys dari komputer dan melihat seluruh list SSH Keys yang telah didaftarkan ke GitLab. Untuk menambahkan SSH Keys user dapat menekan tombol ADD SSH KEY.
