Jadi apa itu Git?

Git : Distributed Version Control System

Bagi kami, dongeng tadi adalah sebuah mimpi buruk. Bayangkan kalau tim Siswa adalah tim yang mengerjakan proyek besar bernilai ratusan juta. Bisa jadi salah satu dari mereka akan kabur ke negeri seberang untuk memulai hidup baru.

Sekarang coba bayangkan kalau ada teknologi yang dapat memudahkan tim Siswa untuk mengerjakan tugas kuliah bersama. File tugas kuliah mereka tersimpan di suatu tempat yang aman. Siswa dapat dengan mudah menggabungkan hasil pekerjaan masing-masing anggota tim, tanpa harus menggunakan flashdisk atau kirim via email. Siswa juga tidak perlu lagi merasa tidak enak untuk melaporkan anggota tim-nya yang tidak atau kurang berkontribusi. Progress pekerjaan dapat dimonitor dan dikontrol dengan mudah tanpa perlu takut kehilangan file karena semua file sudah tersimpan secara online.

Teknologi itu adalah Git. Git melabeli dirinya sebagai Distributed Version Control System. Kata kunci yang perlu diperhatikan disini adalah 'Distributed' dan 'Version'.

Version disini maksudnya adalah Git berfungsi untuk menyimpan banyak versi dari file-file yang di monitor/kontrol. Git menyimpan semua data perubahan dari file-file yang di monitor/kontrol ke sebuah database git. Database perubahan data atau versi file dari git ini disebut Git Repository.

Sedangkan Distributed maksudnya adalah terdistribusi. Semua anggota tim dapat menduplikat satu Git Repository yang sama ke masing-masing komputer, dan bekerja tanpa harus khawatir mengganggu progress pekerjaan anggota tim yang lain.

Kurang lengkap rasanya jika bagian sejarah tidak kami sertakan di sini. Git dibuat oleh Linus Torvald, bagi yang belum tahu siapa beliau, beliau adalah pembuat Kernel Linux. Kernel OS yang sudah dipakai oleh miliaran orang di dunia ini (Android, Ubuntu, Fedora, dsb). Walaupun sebenarnya istilah 'pembuat' ini kurang tepat juga, karena kernel Linux sebenarnya adalah hasil kerja sama banyak programmer dari seluruh penjuru dunia. Penjuru dunia? Semuanya berkumpul di satu gedung yang sama? Tidak. Om Linus lebih sering mengerjakan Kernel Linux dari ruang kerjanya sendirian.

Begitu juga dengan programmer-programmer lain yang berkontribusi. Mereka sebagian besar tidak sering bertatap muka. Semua diatur dengan menggunakan Git. Bahkan Om Linus sekarang jarang sekali coding Kernel Linux, beliau lebih sering menggabung-gabungkan hasil codingan programmer lain ke repository Kernel Linux. Karena itulah Om Linus dijuluki Benevolent dictator for life. Selama Om Linus masih bernafas, beliaulah yang memutuskan apakah hasil codingan programmer lain boleh digabungkan ke kernel Linux atau tidak. Jadi walaupun sudah jarang coding Kernel Linux, Om Linus masih berperan besar dalam menentukan jalan hidup Kernel Linux.

Istilah Benevolent dictator for life ini akan kalian terapkan juga di pelatihan ini. Tapi mungkin disesuaikan jadi Benevolent dictator until assignment submitted.

Sekarang kita akan coba kenali Git lebih dalam. Mungkin masih ada yang belum menangkap Git itu sebenarnya makhluk apa. Kami belum menggunakan istilah-istilah rumit dulu. Implementasinya juga belum kami sertakan di sesi 1 ini.
Belum ada petunjuk atau tutorial harus pencet tombol apa atau input command apa. Yang penting sekarang kalian bisa menangkap cara kerja Git dulu.
